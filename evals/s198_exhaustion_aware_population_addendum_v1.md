# S198 exhaustion-aware population addendum

The first frozen source selection stopped before creating a packet or calling any model:
after all S194+S195+S197, protected-target, source-file and product-pair exclusions, the
generic selector could satisfy seven table manufacturers but found only five eligible prose
manufacturers. The physical scan receipt was not persisted; that instrumentation defect and
the exact observed exception are sealed in `s198_fresh_population_no_go_v1.json`.

This addendum changes only the population-construction envelope. It uses no authored question,
claim, facet, semantic issue, planner result or protected-target result. Therefore it does not
tune against model performance and does not reopen any closed investigation.

The replacement source packet is a new `v2` authority with the same historical exclusions and
the same deterministic seed, but an asymmetric exhaustive stratum target:

- seven table documents/manufacturers;
- all five available prose documents/manufacturers after the table selection;
- twelve unique documents, manufacturers and manufacturer/product pairs in total;
- exact inventory and post-selection reserve persisted before paid execution.

The semantic GO minima do not move: at least 12 eligible items/manufacturers, at least five
table and five prose items, at least 24 accepted points, and zero invalid or semantic failures.
Because the packet contains exactly 12 items, every item must be eligible for GO. Maximum calls
become 12 per stage, 48 paid inferences, 48 token-count preflights and 96 provider requests.
Models, prompts, eligibility, facet precedence, schemas, zero-failure gates, SDKs, prices,
budget, one-shot/no-repair rule and upstream→downstream ordering are unchanged.

This is a new frozen selection attempt, not a retry of the failed 7+7 contract. If 7+5 cannot be
sealed, S198 stops without another population adjustment. No additional frontier-review round is
opened because the semantic package is byte-identical in intent and the change is solely the
measured availability envelope. `chunks_v3` remains `FINAL_NO_GO_CHUNKS_V3_WHOLESALE`, and
Railway remains outside PR/merge gates.
